# Настройка и администрирование бота

В этом разделе описаны основные возможности для пользователей и администраторов Telegram-бота Yadreno VPN.

## Команды бота

Бот поддерживает следующие команды, вы можете их добавить в @BotFather:

| Команда | Описание |
|---------|----------|
| `/start` | Главная страница с приветствием и списком тарифов |
| `/mykeys` | Просмотр ваших VPN-ключей |
| `/help` | Справочная информация о работе с ботом |
| `/update` | ⚙️ Экстренное обновление бота (только для администраторов) Не надо добавлять эту команду в меню бота |

## Основные возможности для пользователей

### 🔑 Покупка и управление ключами
- Покупка VPN-ключей через Telegram Stars или USDT
- Просмотр статуса и трафика ключей
- Продление подписки
- Замена ключа на другой сервер (если использовано менее 20% трафика)
- Переименование ключей для удобства
- **Пробная подписка**: возможность бесплатно протестировать сервис (если включено администратором)

### 💳 Способы оплаты
- **Telegram Stars** - оплата звездами Telegram
- **USDT** - оплата через Ya_SellerBot (https://yadreno.ru/seller/)
- **Банковские карты (ЮКасса)** - оплата картами в рублях.
    
    **Как настроить приём карт:**
    1. ❗️ **РЕГИСТРАЦИЯ:** Первым делом [обязательно зарегистрируйте магазин в ЮКассе по этой ссылке](https://yookassa.ru/joinups/?source=sva) и дождитесь проверки профиля. Без этого прием платежей не заработает.
    2. Перейдите в [@BotFather](https://t.me/BotFather).
    3. Отправьте команду `/mybots` и выберите вашего бота.
    3. Нажмите кнопку **Payments**.
    4. Выберите **YooKassa** (для тестирования «Connect ЮKassa: тест», для работы — «реальные платежи»).
    5. BotFather перенаправит вас в бот провайдера (например, @YooKassa_bot). Нажмите там «Старт», авторизуйтесь в ЮKassa и выберите ваш магазин (ShopID).
    6. **Важно:** После подтверждения привязки в боте провайдера, **вернитесь обратно в @BotFather**.
    7. В @BotFather снова зайдите в `/mybots` → `Ваш бот` → `Payments` → `YooKassa`. 
    8. Теперь BotFather выдаст вам **Provider Token**. Он выглядит примерно так: `123456789:TEST:abcdef...` или `123456789:LIVE:abcdef...`.
    9. В админ-панели нашего бота перейдите: **Админ-меню** → **Оплаты** → **Оплата картами** → **Установить токен** и вставьте полученный ключ.
    
    > ⚠️ **Важно (Лимиты Telegram):** Сервера Telegram (Payments API) имеют жесткий лимит на минимальную сумму платежа — она должна быть эквивалентна **$1 USD**. Из-за плавающего курса, для рублей (RUB) этот порог обычно составляет около 100 руб. Если вы установите цену тарифа меньше этого лимита, оплата картой не пройдет, а пользователи увидят уведомление об ошибке.

### 📊 Информация о ключах
- Статус подключения (активен/истек)
- Использованный трафик
- Дата окончания подписки
- Сервер и протокол подключения

### 👥 Реферальная система
Пользователи могут приглашать друзей и получать вознаграждение:
- **Реферальная ссылка** — уникальная ссылка для приглашения
- **Многоуровневые бонусы** — вознаграждение до 3 уровней рефералов
- **Два типа вознаграждения** (зависит от настроек администратора):
  - 📅 **Дни к ключу** — автоматическое продление подписки
  - 💰 **На баланс** — накопление средств для оплаты
- **Статистика** — просмотр количества рефералов и заработанного вознаграждения по каждому уровню

## Для администраторов

Администраторы имеют доступ к дополнительным функциям через кнопку "⚙️ Админ-панель":
- Управление серверами
- Настройка тарифов (включая лимиты трафика)
- Управление пользователями (поиск по `telegram_id`, `@username` или `panel_email` из панели 3X-UI)
- **Управление пробной подпиской** (вкл/выкл, выбор тарифа)
- **Обновление бота** (обычные обновления, блокирующие и бета-версии)

- Редактирование текстов бота (главная, справка, ссылки)
- Рассылка сообщений
- **Скачивание логов** (полного лога работы бота или отдельного файла с зафиксированными ошибками)
- Просмотр статистики

### 🔄 Система обновлений

В разделе **Настройки бота** -> **Обновления** администратор может обновлять код бота прямо из GitHub репозитория.

Виды обновлений:
1. **Обычные обновления** — применяются по нажатию кнопки «Обновить и перезапустить». О выходе таких обновлений бот автоматически уведомляет администраторов (если включён планировщик).
2. **Блокирующие обновления (`!`)** — если сообщение коммита начинается с `!`, бот предупредит, что это важное обновление (например, требующее миграции БД). При нажатии «Обновить до блокирующего коммита» бот установит именно этот коммит.
3. **Бета-версии (`?`)** — если сообщения всех новых коммитов начинаются с `?`, бот **не будет** рассылать уведомления администраторам. Вы можете зайти в раздел обновлений и нажать специальную кнопку **«Накатить бета версию»**, чтобы протестировать новые функции на свой страх и риск.

Если что-то пошло не так, можно воспользоваться кнопкой **«Принудительно перезаписать»** — она скачает самую последнюю версию из ветки и затрёт все локальные изменения в коде (база данных при этом не затрагивается).

### 📊 Система учёта трафика

Бот поддерживает ограничение пользователей по объему трафика:
- **Установка лимитов**: При создании/редактировании тарифа вы можете указать лимит в ГБ. Если указать `0` — трафик будет безлимитным.
- **Остаток при переезде**: Если пользователь меняет сервер в боте, его неизрасходованный остаток трафика бережно переносится на новый сервер.
- **Ежемесячный сброс**: В разделе **Админ-меню** → **Оплаты** есть переключатель **«🔄 Автосброс трафика 1-го числа»**. Если он включен, то 1-го числа каждого месяца всем пользователям обнуляется израсходованный трафик, и им снова доступен полный лимит по их тарифу.
- **Уведомления**: Бот сам предупредит клиента, когда у того останется 10%, 5%, 3% и менее процентов трафика.

### 🔄 Синхронизация данных с панелью

Бот является **единственным источником правды** для дат истечения ключей и лимитов трафика. Данные на панели 3X-UI автоматически синхронизируются с базой бота.

**Автоматическая сверка (1-е число каждого месяца):**
Бот автоматически сравнивает данные на панелях с базой данных. Если обнаружены расхождения — бот исправляет данные на панели и отправляет отчёт администраторам.

**Ручная синхронизация:**
Перейдите: **Админ-меню** → **Пользователи**

| Кнопка | Описание |
|--------|----------|
| **📤 Выгрузить в панель (БД → Панель)** | Записывает даты истечения и лимиты трафика из БД бота на панели 3X-UI. Используйте, если данные в боте — верные, а на панели устарели. |
| **📥 Загрузить из панели (Панель → БД)** | Считывает данные с панелей 3X-UI и обновляет БД бота (даты, лимиты, трафик). Используйте, если данные на панели — верные, а в боте устарели. |

Обе кнопки покажут отчёт: сколько ключей обновлено, сколько без расхождений, ошибки.

> 💡 Используйте **📤 Выгрузить**, если вы редактировали данные в боте и хотите применить их на панели. Используйте **📥 Загрузить**, если вы вносили изменения напрямую на панели 3X-UI и хотите синхронизировать их обратно в бот.

### 🔗 Реферальная система

#### Настройка

Перейдите: **Админ-меню** → **Реферальная система**

| Параметр | Описание |
|----------|----------|
| **Включить/Выключить** | Глобальное включение или отключение системы |
| **Режим начисления** | 📅 Дни к ключу — добавляются дни к VPN-ключу реферера. 💰 На баланс — начисляются рубли на внутренний баланс |
| **Уровни 1-3** | Настройка процента (1-100%) и включение каждого уровня. Уровень 1 — прямой реферал, 2 — реферал реферера, 3 — выше по цепочке |
| **Текст условий** | Текст с условиями программы для показа пользователям |

> ⚠️ При оплате **балансом** реферальные вознаграждения НЕ начисляются.

#### Управление балансом и коэффициентами

В карточке пользователя (**Пользователи** → выбор пользователя):

| Действие | Описание |
|----------|----------|
| **Пополнить / Списать** | Ручное изменение баланса пользователя |
| **Коэффициент** | Индивидуальный множитель вознаграждения (0.0x - 10.0x, по умолчанию 1.0x) |

> 💡 Пример: коэффициент 2.0x при уровне 10% → при оплате рефералом 1000 ₽ пользователь получит 200 ₽ вместо 100 ₽.

### 📂 Группы тарифов и серверов

Механизм групп позволяет изолировать серверы и тарифы друг от друга. По умолчанию все элементы попадают в группу "Основная".

**Основные принципы:**
- **Создание группы**: В Админ-меню → Оплаты → Группы тарифов можно добавить новую группу.
- **Привязка**: При создании нового тарифа или сервера бот предложит выбрать, к какой из групп они будут привязаны.
- **Изоляция**: Пользователи, купившие ключ с тарифом из определенной группы, смогут продлевать его только тарифами из этой же группы. При замене сервера им будут доступны только серверы из их группы.
- **Отображение**: Если создана больше одной группы (и в ней есть хотя бы 1 активный тариф и 1 сервер), они будут визуально разделены на категории в меню покупки для пользователей.

### 📝 Редактирование текстов
В админ-панели можно менять тексты и ссылки без перезагрузки бота:
1. **Текст главной страницы**: Приветствие при команде `/start`.
2. **Текст справки**: Содержимое раздела `/help`.
3. **Текст перед оплатой**: Сообщение с условиями и отказом от ответственности, которое показывается пользователю перед выбором способа оплаты.
4. **Ссылки**: Настройка URL для кнопок «Новости» и «Поддержка».

#### Специальные теги для текста главной страницы

В тексте главной страницы можно использовать теги для управления выводом списка тарифов:

| Тег | Поведение |
|-----|-----------|
| *(тег не указан)* | Список активных тарифов **автоматически добавляется в конец** сообщения |
| `%тарифы%` | Список тарифов вставляется **в указанное место** текста |
| `%без_тарифов%` | Тарифы **не выводятся** совсем (тег удаляется из текста) |

> 💡 **Пример:** Чтобы разместить тарифы между двумя абзацами, вставьте `%тарифы%` в нужное место. Если тарифы вообще не нужны на главной — добавьте `%без_тарифов%` в любом месте текста.
